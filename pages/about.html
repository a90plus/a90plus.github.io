About a90plus

Design philosophy, data sources, and how to contribute.

The Per-Cup Philosophy

Every piece of data on a90plus is scoped to a single World Cup edition. Messi's 2022 goals are stored only in 2022.json. Argentina's 1978 title lives only in 1978.json. There are no stored all-time totals, no career goals fields, no lifetime aggregates.

Any cross-tournament view you see (e.g. "all-time top scorers") is computed at query time by summing across the per-cup files. The data layer stays clean and tournament-correct. Historical nations like West Germany (FRG) are never merged with modern Germany (DEU).

Event-Level Granularity

Goals, assists, cards, substitutions, and VAR decisions are all stored as typed events — not as summary numbers. A player's goal count is always derived by counting type: "goal" events. The match's events[] array is the single source of truth.

Data Sources

  • FIFA Official Match Reports
  • Wikipedia FIFA World Cup articles
  • Transfermarkt player bios
  • RSSSF (Rec.Sport.Soccer Statistics Foundation)
  • Natural Earth (map geometry, public domain)

Current data files are marked isMock: true, verified: false. Figures may contain inaccuracies. Please verify against primary sources before citing.

Tech Stack

  • Plain HTML + CSS + vanilla JavaScript — no build step, no framework
  • D3.js v7 + TopoJSON for the choropleth map (CDN)
  • Google Fonts (Inter) via CDN
  • Data served via jsDelivr CDN from the a90plus/a90plus-data GitHub repo
  • Hosted on GitHub Pages at a90plus.github.io
  • Python 3.11 + jsonschema + pytest for data tooling

Contributing

Data contributions are welcome. To add or correct a tournament:

  1. Fork a90plus/a90plus-data
  2. Edit or create tournaments/<year>.json
  3. Run python tools/validate.py tournaments/<year>.json
  4. Open a Pull Request — the CI workflow validates automatically

Set verified: true only when all data has been cross-checked against official FIFA match reports.
